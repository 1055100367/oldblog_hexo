Mac连接Android

2019-04-09

今日闲着没事干把一个老安卓设备翻出来折腾….

远程

软件使用开源项目scrcpy

需要Android API>=21

需要开启ADB

安装Scrcpy

Windows

去Releases下载对应版本解压就vans了

Linux

wiki有说咋安装,因为本人没实践过,所以就在此就不写出来了

MacOS

brew install scrcpy

如果安装时出现如下信息

则执行xcode-select —install后重新安装

运行scrcpy

在已连接(USB/Wi-Fi)Android后执行scrcpy即可控制安卓设备

安装ADB(参考xda帖子)

Windows

下载ADB(需要”特殊”网络)

度盘

解压后使用powershell/cmd进入该文件目录,或在解压后的文件目录按住shift+右键在此打开命令行

配置path的话把目录添加进PATH环境变量即可

安卓设备开启USB调试,确认已连接安卓设备后adb devices即可看到设备信息

Linux

Debian系:sudo apt install android-tools-adb -y

Redhat系:sudo yum install android-tools-adb -y

Arch系:去ArchWiki叭ovo

安卓设备开启USB调试,确认已连接安卓设备后adb devices即可看到设备信息

MacOS

brew cask install android-platform-tools

安卓设备开启USB调试,确认已连接安卓设备后adb devices即可看到设备信息

Q&A

Q:command not found

A:检查是否已安装adbwhich adb,检查环境变量是否配置

Q:连上手机看不到设备

A:检查USB调试是否开启,检查数据线连接是否正常,检查驱动是否正常安装,检查Vendor ID是否已添加到~/.android/adb_usb.ini里

SSH

如果觉得单纯控制手机不爽的话,可以ssh到Android上蹂躏机器人

Android安装termux可在酷安/Google Play/F-Droid等平台下载

进入终端后apt update apt install sshd

使用passwd修改当前用户登录密码

使用whoami查看当前用户名

在已连接(USB/同一Wi-Fi)Android情况下ssh username@ip -p 8022
